# IRIAM-US 業務相談・意思決定ログ リポジトリ

## このリポジトリについて

IRIAM-US事業責任者がAIと業務相談を行い、コンテキストと意思決定を蓄積するためのリポジトリ。
日々のオペレーションから戦略的意思決定まで幅広く活用する。

---

## Claudeへの行動指針

### 【必須】会話の開始時に必ず行うこと

以下のファイルを読んでから応答を始める:
- `context/company.md` - 事業・競合・戦略
- `context/product.md` - プロダクト・指標・施策
- `context/team.md` - チーム・役割
- `context/constraints.md` - 制約条件
- `tasks/current.md` - 現在のアクティブタスク

### 新しい情報・数値が出てきたとき

1. **即座に該当の `context/` ファイルを更新する**
   - KPI数値の更新、施策の進捗、新たな事実など
   - 「後で更新する」はしない。会話の中でリアルタイムに更新する
2. **既存の記録と矛盾する場合は必ず確認してから更新する**
   - 例:「先月の採用数が70名と記録されていますが、今おっしゃった数値と異なります。どちらが正しいですか？」

### 意思決定・重要な判断が出てきたとき

- `decisions/YYYY-MM-DD_[テーマ].md` の作成を提案する
- テンプレート: `templates/decision-record.md`
- 明確な「決定」でなくとも、重要な方針変更・優先順位の変更があれば提案する

### 施策の進捗・検証・振り返りが出てきたとき

- `topics/[領域]/YYYY-MM-DD_[テーマ].md` への記録を提案する
- テンプレート: `templates/problem-analysis.md`
- 領域の例: `marketing/`, `streamer-acquisition/`, `arpu/`, `ops/`

### タスクの追加・更新・完了が出てきたとき

- **新しいタスクが出てきたとき**: 種別に応じて `tasks/current.md` の適切なセクションに追加する
  - 判断が必要なもの → 🔴 要意思決定
  - 自分や誰かが動かすもの → 🟡 進行中
  - 誰かへの確認待ち → 🟢 フォローアップ待ち
  - 近日中にやること → ⬜ 今週・近日中
  - 優先度未確定 → `tasks/backlog.md` に追加
- **タスクが完了したとき**: `tasks/current.md` から削除し、当日のデイリーログに記録を残す
- **デイリーログ**: 会話の終わりに「今日のまとめを `tasks/daily/YYYY-MM-DD.md` に記録しますか？」と提案する
  - テンプレート: `templates/daily-log.md`

### 過去ログとの整合性チェック

- 以前の議論や記録と矛盾する情報が出た場合は、流さずに指摘する
- 「以前〇〇と記録されていましたが、今の話と整合していますか？」と確認する

## 情報の種類と保存先の判断基準

| 情報の種類 | 保存先 | 例 |
|-----------|--------|-----|
| 変わらない背景・前提・方針 | `context/` | KPI定義、用語、ビジネスモデル |
| 変化する数値・状況 | `context/` を更新（最新値に上書き） | DAU、採用数、出稿額 |
| 数値の推移・時系列 | `topics/` に追記 | 月次KPI推移、施策効果 |
| 明確な意思決定 | `decisions/` に新規作成 | 施策採択、方針変更 |
| 検討中・仮説段階 | `context/` に「仮説」として記載 or `topics/` | 未確定の分析 |
| アクティブなタスク | `tasks/current.md` を更新 | 進行中の施策、フォローアップ |
| 優先度未確定のタスク | `tasks/backlog.md` に追加 | 将来候補、検討中 |
| 当日の更新・完了記録 | `tasks/daily/YYYY-MM-DD.md` | 完了タスク、メモ |

## 用語・略語クイックリファレンス

| 用語 | 定義 |
|------|------|
| DAU | デイリーアクティブユーザー数 |
| DLU | Daily Liver User（1日あたりの配信者数） |
| NUU | New Unique User（新規ユーザー数） |
| CPI | Cost Per Install（1インストールあたりの広告費用） |
| ROI | Return on Investment（投資対効果） |
| Native EAS | デビューサポートプログラム経由でデビューするストリーマー |
| EX-EAS | 外部プラットフォームからIRIAMに来るストリーマー（目標数に含まない） |
| ライバー | 配信者 |
| リスナー | 視聴者 |
| ギフティング | 視聴者からライバーへの投げ銭 |
