# AI業務活用 コンテキスト & 意思決定ログ

AIとの業務相談の精度を高め、知識を組織資産として蓄積するためのリポジトリ。

---

## 目的

- **再現性**: 毎回同じ背景説明をしなくて済むようにする
- **継続性**: 検討の経緯と結論を残し、後から参照・活用できるようにする
- **精度向上**: 蓄積したコンテキストをAIに渡すことで、より的確なアドバイスを得る

---

## ディレクトリ構成

```
├── context/        # AIに渡す背景情報（常に最新を維持）
├── decisions/      # 意思決定ログ（1決定 = 1ファイル）
├── topics/         # テーマ別の継続的な相談ログ
├── templates/      # 再利用できるプロンプト・ドキュメントテンプレート
└── README.md       # このファイル
```

---

## 使い方

### AIに相談するとき
1. `context/` の関連ファイルをAIに渡す（コピペ or ファイル添付）
2. 相談・検討を行う
3. 結論が出たら `decisions/` または `topics/` にログを残す
4. 新たに判明した背景情報は `context/` を更新する

### 意思決定を記録するとき
`templates/decision-record.md` をコピーして `decisions/YYYY-MM-DD_[テーマ].md` として保存する。

### 問題分析を行うとき
`templates/problem-analysis.md` をコピーして `topics/[領域]/YYYY-MM-DD_[テーマ].md` として保存する。

---

## ファイル命名規則

| 種類 | 形式 | 例 |
|------|------|----|
| 意思決定ログ | `YYYY-MM-DD_[キーワード].md` | `2026-02-25_pricing-strategy.md` |
| テーマ別ログ | `YYYY-MM-DD_[キーワード].md` | `2026-02-10_competitor-analysis.md` |

---

## 更新ルール

- `context/` のファイルは情報が変わったら**即時更新**する（日付ベースではなく常に最新状態）
- 過去の決定は**削除しない**（状態を「廃止」に変更する）
